# Continuity — Always Active

This rule has no path scope and is always loaded. Continuity is the connective tissue of a 9-book series.

## Core Principle

The bible is canonical. When in doubt, check the bible. When the bible and a draft disagree, the bible wins.

## Precedence

Earlier books take precedence over later books in all factual disputes. A fact established in Book 1 cannot be contradicted in Book 7 without an explicit, in-story explanation.

## Common Continuity Traps

Watch for these across every chapter:

- **Character ages**: Characters age in real time. If Book 1 is set in 1340 and a character is 25, they're 35 in a Book 3 set in 1350.
- **Scars and injuries**: If a character is wounded in Book 2, they have that scar in every subsequent book. Check `Continuity Notes` in their profile.
- **Knowledge**: Characters only know what they've been told or witnessed. Don't let a character reference an event they weren't present for.
- **Travel times**: Moving between locations takes time. Check `bible/world/locations.md` and `technology-and-daily-life.md` for realistic travel durations.
- **Relationships**: Characters' feelings about each other are informed by their shared history. Check the relationship map before writing interactions.
- **Seasonal details**: If the timeline says it's winter, the landscape should reflect that. Weather, clothing, available food.
- **Promises and oaths**: If a character swore an oath or made a promise, track it. Readers remember.
- **Death**: Dead characters stay dead (unless the story has a mechanism for otherwise, which must be established early).
- **Skills and abilities**: Don't give characters skills they haven't been established to have. Check their background.

## When You Spot an Issue

1. **Never silently resolve it.** Don't just change the draft or the bible to make things fit.
2. Add a `<!-- FLAG: CONTINUITY — [description] -->` comment in the draft.
3. Log it in the book's `notes/continuity-notes.md`.
4. If it affects the bible, note it but don't change the bible without explicit approval.
